FE-SERVICE Schritt 2

Wichtig: Wenn Tickets trotz Codeänderung nicht speichern, liegt es fast sicher an Supabase RLS.

1. ZIP entpacken und Dateien ins Projekt kopieren.
2. Datei supabase-ticket-fix.sql öffnen.
3. In Supabase einloggen.
4. SQL Editor öffnen.
5. Inhalt von supabase-ticket-fix.sql einfügen und Run drücken.
6. Danach lokal testen:

npm install
npm run dev

Wenn du Vercel nutzt: Danach neu deployen.
